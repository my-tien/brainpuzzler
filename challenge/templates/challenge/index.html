SegEM Challenge

The SegEM Challenge is an open competition to find the most suitable machine learning algorithms that segments 3D EM datasets. The competition is held by the Max Planck Institute for Medical Research located in Heidelberg, Germany.

In our segmentation cube format, we use 64bit matrices with an edge length of 128 voxels. These cubes are compressed with Google snappy, and should have the file type .seg.sz.

The segmentation should be wrapped in a KNOSSOS annotation file (.k.zip) to submit it. In our GitHub repository , you will find Python code for generating such files.

Due to the large size of the dataset, our evaluation is based on a sparse skeleton that was obtained by aggregating thousands of manual tracings made by students. We compare your submission against this skeleton and count the number of correctly as well as incorrectly associated locations to determine your score.
